# Impeccable

## Design Context

Two design surfaces share one brand. Both feel like the terminal app; they differ in job, not in family.

### Shared Brand

- **Palette**: warm dark background (#1A1816), warm off-white text (#E8E4DC), yellow accent (#F9C844), blue links (#82BAD5).
- **Fonts**: Atkinson Hyperlegible Mono (headings/UI), Inter (body), 0xProto (code).
- **Personality**: friendly, witty, no-nonsense (README: "some of us struggle with a single brain already!"). Casual jokes welcome, but every sentence earns its place. Confident without smug; helpful without hand-holding.
- **Anti-reference**: corporate SaaS (Stripe/Notion-style sterile gradients, marketing speak). Nothing should feel like a startup landing page.

---

### Surface: Docs site (`docs/`)

#### Users
Developers and terminal power users who keep notes in plain Markdown. They live in the shell, value speed and ownership of their data, and read docs to get unstuck fast — installation, keybindings, CLI flags, search syntax. Many arrive from the GitHub README already sold on the "lightweight, local-first" pitch.

#### Design Principles
1. **Scannable over thorough** — tables, short sections, and examples beat prose walls. A user should find a keybinding in under 10 seconds.
2. **Show, don't explain** — lead with a command or screenshot, explain after.
3. **Feel like the terminal** — monospace flavor, TUI palette, no decorative web fluff.
4. **Wit in small doses** — one good aside per page, not a comedy routine.
5. **Local-first pride** — copy reinforces ownership: your files, your machine, no lock-in.

---

### Surface: Server admin web UI (`server/src/webui.rs`)

#### Users
The self-hosting vault owner administering their own RAG server. Occasional visits, not a daily driver: glance at running jobs, check a collection, tweak config, fire a test query, leave. Single user, own machine — the UI answers "is my server fine?" fast and gets out of the way.

#### Feel
Calm instrument panel with terminal-native charm. Quiet confidence — numbers steady, nothing shouting — but unmistakably part of Kimün, not a generic admin template. Small asides in the docs-site voice are welcome (one per page, max).

#### Aesthetic Direction
Match the docs/TUI family: warm dark terminal palette and shared type voice. Dark is the only theme — the audience lives in dark terminals. Yellow accent stays rare (active nav, primary action); blue for links. Tint neutrals toward the warm brand hue, never cool gray.

#### Technical Constraints
- Server-rendered maud templates, inline CSS, sprinkle of inline JS for job polling. No build step, no node, no external requests.
- **Fonts are embedded in the binary**: ship woff2 bytes served from a route (`include_bytes!`), not fetched from a CDN. Subset if size matters.
- Everything must work from a single self-contained binary.

#### Design Principles
1. **Glanceable state first** — the dashboard answers "is my server healthy, what's indexing" before anything else; jobs and counts readable at arm's length.
2. **One family, three surfaces** — TUI, docs, and admin UI share palette and type; a user moving between them should never feel a seam.
3. **Instrument panel, not marketing** — dense tabular data, tabular-nums, honest labels; no hero metrics, no decorative charts.
4. **Accent is a signal** — yellow means "active" or "act here"; if it appears more than a few times per page it has stopped meaning anything.
5. **Config edits are deliberate** — settings persist to TOML and apply on restart; the UI says so plainly and never pretends changes are live.
