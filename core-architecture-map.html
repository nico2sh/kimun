    Cartographer's notes — invariants that shape this map

    • All file ops live in core; the TUI never touches paths or files.
    • Direct filesystem calls exist only inside nfs.
    • Public API speaks VaultPath, never PathBuf, for vault-internal paths.
    • Paths are case-insensitive, lowercased, and valid on all three major OSes.
    • Per-note locks acquired in stable sorted order → deadlock-free read-modify-write.
    • Destructive ops (save / delete / replace / rename) back up pre-images when enabled.
    • Index is a rebuildable cache — the filesystem is always the source of truth.
    • Delete order: index first, filesystem second. Create order: filesystem first, index second.
